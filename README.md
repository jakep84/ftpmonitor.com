# 🚀 FTPMonitor

![npm](https://img.shields.io/npm/v/ftpmonitor)
![npm downloads](https://img.shields.io/npm/dm/ftpmonitor)
![license](https://img.shields.io/npm/l/ftpmonitor)

**Instant diagnostics for FTP, FTPS, and SFTP endpoints.**

Stop guessing why file transfers fail.

FTPMonitor runs structured diagnostics that reveal exactly where a connection breaks:

DNS → TCP → Authentication → Directory Listing

Designed for **DevOps engineers, sysadmins, developers, and support teams** who need answers fast.

---

# ⚡ Try It Instantly (Zero Install)

Run a real diagnostic in seconds:

```bash
npx ftpmonitor check --protocol sftp --host example.com --username user
```

Paste-ready output for Slack, Jira, or tickets:

```bash
npx ftpmonitor check --protocol sftp --host example.com --username user --format slack --copy
```

CI-friendly JSON:

```bash
npx ftpmonitor check --protocol sftp --host example.com --username user --format json
```

# 🔎 What FTPMonitor Checks

Every run performs structured diagnostics:

1. **DNS Resolution**
2. **TCP Connectivity**
3. **Authentication**
4. **Directory Listing**

Each step includes:

• Success / failure state  
• Timing information  
• Diagnostic messages  
• Structured output for automation

# 🤖 CI / Automation

FTPMonitor works well inside CI pipelines.

Example:

```bash
npx ftpmonitor check   --protocol sftp   --host example.com   --username user   --format json
```

Exit codes:

```
0 → Success
2 → One or more checks failed
3 → Usage or validation error
```

---

# 🌍 Hosted API

Run checks remotely using the FTPMonitor hosted API.

```bash
ftpmonitor check   --api https://ftpmonitor.com/api/health-check   --protocol sftp   --host example.com   --username user   --password pass
```

Useful when the target endpoint must be tested from a specific network.

# 🧠 Why This Exists

FTP is still heavily used in:

• Healthcare data exchange  
• Financial batch processing  
• Government vendor integrations  
• Vendor file drop systems  
• Legacy B2B pipelines

When these fail, diagnosing the issue is usually slow and manual.

FTPMonitor provides a **fast, structured way to identify the exact failure point**.

# 🛠 Development

Clone the repository:

```bash
git clone https://github.com/jakep84/ftpmonitor.com
cd ftpmonitor.com
```

Install dependencies:

```bash
npm install
```

Run the web app:

```bash
npm run dev
```

Build the CLI:

```bash
cd packages/ftpmonitor
npm run build
```
